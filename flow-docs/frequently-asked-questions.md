# <a name="frequently-asked-questions"></a>Domande frequenti

## <a name="audience-and-strategy"></a>Destinatari e strategia
### <a name="what-is-microsoft-flow"></a>Che cos'è Microsoft Flow?
Microsoft Flow è un servizio basato sul cloud che rende pratico e semplice per gli utenti line-of-business creare flussi di lavoro che automatizzano le attività e i processi aziendali che richiedono molto tempo tra applicazioni e servizi.

### <a name="who-is-the-intended-audience-for-microsoft-flow"></a>Chi è il pubblico previsto per Microsoft Flow?
Microsoft Flow dispone di due destinatari distinti:

* "Integratori di cittadini" line-of-business nelle organizzazioni aziendali che collaborano con esso per trasferire la responsabilità per le soluzioni aziendali più vicine all'azienda stessa.
* Responsabili decisionali IT che desiderano consentire ai partner line-of-business di creare soluzioni personalizzate in modo che i professionisti IT e gli specialisti dell'integrazione possano concentrare le proprie competenze su strumenti di integrazione più avanzati, come le app per la logica di Azure.

### <a name="how-do-microsoft-flow-and-logic-apps-relate-to-each-other"></a>In che modo le app per la logica e Microsoft Flow sono correlate tra loro?
Microsoft Flow fornisce funzionalità che consentono agli utenti line-of-business di creare flussi di lavoro automatizzati. App per la logica è un servizio di Azure che offre le stesse eccezionali funzionalità di Microsoft Flow, oltre a funzionalità quali l'integrazione con Azure Resource Manager e il portale di Azure, PowerShell e l'interfaccia della riga di comando di xPlat, Visual Studio e altri connettori. [Altre informazioni sulle app per la logica](https://azure.microsoft.com/services/app-service/logic/).

### <a name="how-does-microsoft-flow-fit-in-microsofts-overall-business-application-platform-strategy"></a>In che modo Microsoft Flow si adatta alla strategia globale per le applicazioni aziendali di Microsoft?
Microsoft Flow fa parte di una piattaforma applicativa aziendale potente e adattabile che include PowerApps, il Common Data Service, Dynamics 365 e Office 365. Questa piattaforma consente ai clienti, ai partner e ai partner ISV di creare soluzioni progettate appositamente per le proprie aziende, il proprio settore, per i ruoli funzionali o anche per aree geografiche specifiche. Gli utenti line-of-business, che conoscono meglio le proprie esigenze aziendali, ora possono analizzare, comporre e semplificare i dati e i processi in modo semplice. Gli sviluppatori professionisti possono estendere facilmente l'automazione, le analisi e le app line-of-business per sfruttare i servizi di Azure, come funzioni, servizio app e app per la logica. Connettori API, gateway e Microsoft Common Data Service consentono di sfruttare al meglio i servizi o i dati già in uso, sia nel cloud che in locale.

## <a name="functionality"></a>Funzionalità
### <a name="what-do-i-need-to-use-microsoft-flow"></a>Cosa è necessario usare Microsoft Flow?
Per usare Microsoft Flow, è sufficiente un Web browser e un indirizzo di posta elettronica.

### <a name="what-browsers-and-devices-can-i-use-with-microsoft-flow"></a>Quali browser e dispositivi è possibile usare con Microsoft Flow?

È possibile eseguire Microsoft Flow su tutti i dispositivi e i browser moderni.

#### <a name="supported-devices"></a>Dispositivi supportati

Microsoft Flow viene eseguito sui dispositivi moderni. Se è necessario gestire Microsoft Flow da un dispositivo mobile, provare l'app Microsoft Flow per dispositivi mobili disponibile in [iPhone](https://itunes.apple.com/app/microsoft-flow/id1094928825?ls=1&mt=8), [Android](https://play.google.com/store/apps/details?id=com.microsoft.flow)e [Windows Phone](https://www.microsoft.com/p/microsoft-flow/9nkn0p5l9n84?rtc=1#activetab=pivot:overviewtab).

#### <a name="supported-browsers"></a>Browser supportati

Si consiglia di usare il browser più aggiornato compatibile con il sistema operativo. Sono supportati i browser seguenti:

* Microsoft Edge
* Internet Explorer 11
* Safari
* Chrome
* Firefox

### <a name="which-email-addresses-are-supported"></a>Quali indirizzi di posta elettronica sono supportati?
Microsoft Flow supporta gli indirizzi di posta elettronica che terminano con qualsiasi elemento eccetto. gov e. mil.  

### <a name="is-microsoft-flow-available-on-premises"></a>Microsoft Flow è disponibile in locale?
Microsoft Flow è solo un servizio cloud pubblico. Tuttavia, è possibile connettersi in modo sicuro ai propri servizi locali tramite il gateway dati locale.

### <a name="what-services-can-microsoft-flow-connect-to"></a>Quali servizi possono Microsoft Flow connettersi?
Microsoft Flow si connette a più di 100 origini dati predefinite e si sta aggiungendo sempre più tempo. Di seguito sono riportati alcuni esempi di origini dati e servizi:

* SharePoint
* Dynamics 365
* OneDrive
* OneDrive for business
* Google Drive
* Fogli Google
* Trello
* Twitter
* Dialogo
* Facebook
* SalesForce.com
* MailChimp
* API dei clienti

[Qui](https://go.microsoft.com/fwlink/?LinkId=832211)è possibile trovare un elenco completo dei connettori disponibili.

È possibile accedere alle origini dati nella propria infrastruttura IT tramite il [gateway dati locale](gateway-manage.md).

### <a name="what-are-templates"></a>Che cosa sono i modelli?
I modelli sono flussi predefiniti per scenari diffusi e comuni. L'uso di un modello richiede solo l'accesso ai servizi nel modello e la compilazione di tutte le impostazioni necessarie.

### <a name="what-data-sources-will-i-be-able-to-connect-to"></a>A quali origini dati sarà possibile connettersi?
È possibile connettersi a più di 100 servizi standard di Microsoft e di terze parti, ad esempio Office 365, Twitter, SharePoint, OneDrive, Dropbox, SQL Server e altro ancora. È anche possibile connettersi a servizi Premium, ad esempio Salesforce e il Common Data Service per PowerApps.

### <a name="how-do-i-connect-to-a-rest-api-in-my-flow"></a>Ricerca per categorie connettersi a un'API REST nel flusso?
È possibile connettersi a qualsiasi API REST che usa JSON e supporta almeno uno degli oltre 10 metodi di autenticazione creando [un connettore personalizzato](developer/register-custom-api.md).

### <a name="how-do-i-connect-to-sql-server-and-other-on-premises-data-sources"></a>Ricerca per categorie connettersi a SQL Server e ad altre origini dati locali?
È possibile connettersi ai servizi nella rete locale usando il [gateway dati](gateway-manage.md)locale.

### <a name="can-i-share-the-flows-i-create"></a>È possibile condividere i flussi creati?
È possibile condividere i flussi in uno dei modi seguenti:

* È possibile aggiungere colleghi o gruppi all'interno dell'organizzazione come proprietari nei flussi, in modo da poter anche modificare e gestire il flusso.
* Per i flussi che possono essere eseguiti manualmente, è anche possibile concedere ad altri utenti o gruppi nell'organizzazione l'autorizzazione a eseguire semplicemente il flusso.

### <a name="how-many-flows-can-i-have"></a>Quanti flussi è possibile avere?
Microsoft Flow viene fornita con un massimo di 50 flussi. Se necessario, è possibile richiederli.

### <a name="can-flows-be-turned-off-or-disabled"></a>I flussi possono essere disattivati o disabilitati?

Sì, ogni flusso ha un'opzione di attivazione/disattivazione che consente di arrestare l'elaborazione delle richieste da parte del flusso.

Vedere la tabella seguente per comprendere il modo in cui il flusso risponde quando viene riattivato.

Tipo di trigger|Descrizione
-------|--------
Polling, ad esempio il trigger di **ricorrenza**|Quando il flusso viene nuovamente attivato, vengono elaborati tutti gli eventi non elaborati/in sospeso. Eliminare il flusso se non si vuole elaborare gli elementi in sospeso.
Webhook|Quando il flusso viene nuovamente attivato, elabora solo i nuovi eventi che vengono generati dopo che il flusso è stato attivato.

### <a name="what-regions-and-languages-does-microsoft-flow-support"></a>Quali sono le aree e i linguaggi supportati Microsoft Flow?
Microsoft Flow è disponibile in lingue 42 e [sei aree](regions-overview.md).

### <a name="how-does-microsoft-flow-compare-to-sharepoint-designer-2013"></a>In che modo Microsoft Flow viene confrontato con SharePoint Designer 2013?
Microsoft Flow è il successore di SharePoint Designer per molti scenari aziendali comuni, come approvazioni, revisione dei documenti e onboarding/offboarding. Sarà lo strumento predefinito per la creazione di automazione aziendale in SharePoint in futuro.

### <a name="how-does-microsoft-flow-ensure-that-corporate-data-isnt-accidentally-released-to-social-media-services"></a>In che modo Microsoft Flow garantisce che i dati aziendali non vengano accidentalmente rilasciati ai servizi social media?
Gli amministratori possono creare [criteri di prevenzione della perdita dei dati](prevent-data-loss.md) per garantire che vengano usati solo i servizi approvati in Microsoft Flow.

### <a name="does-microsoft-flow-support-service-accounts"></a>Microsoft Flow supporta gli account del servizio?

Sebbene sia possibile creare flussi con un account del servizio, questa operazione non è consigliabile se le credenziali per l'account del servizio sono condivise.
